# 장면 분할 — 알고리즘·파라미터·병렬 설계

## 개요

scenedetect **ContentDetector** 로 콘텐츠 변화 기반 동적 경계를 잡고, 후처리로
정수 초 스냅·병합·분할해 **틈·겹침 없는 시간 타일**을 만든다. 고정 그리드가 아니다.

```
원본 → [탐지] 인접 프레임 HSV 변화 점수 > threshold ⇒ 컷
     → [후처리] min_sec 미만 병합 → max_sec 초과 균등분할 → 정수 초 스냅
     → (seg_id, start, end) 타일
```

## 파라미터 의미

### PREP_THRESHOLD (기본 15.0 운영)

인접 두 프레임의 HSV 픽셀 평균 차이 점수(0~255 스케일)가 이 값을 넘으면 컷.
- 낮추면(15) 작은 변화도 컷 → 촘촘. 높이면(27, 라이브러리 기본) 확실한 전환만.
- **약점**: 페이드/디졸브처럼 서서히 변하는 전환은 점수가 임계값을 안 넘어 놓칠 수 있다.
  `PREP_MAX_SEC` 상한 분할이 피해를 세그 하나로 가둔다. 디졸브가 많은 콘텐츠면
  AdaptiveDetector 병용을 검토.
- 실측(115분 야구 중계): threshold 15→27 에서 세그 1678→1448. 기존 검증본과의 경계
  일치율(F1)은 71~74%로 전 구간 평평 — threshold 로는 못 좁히는 구조적 차이
  (강제분할 위치·최소 간격 정책)가 ~25% 존재한다.

### PREP_MIN_SEC (기본 1.0)

두 역할을 겸한다(별도 프레임 단위 설정 없음 — fps 무관 일관성):
1. 탐지 단계 최소 컷 간격: `min_scene_len = round(min_sec × 영상 fps)` 로 자동 유도.
2. 후처리 병합: min_sec 미만 세그는 직전 세그에 흡수(첫 세그는 다음 세그가 흡수).

### PREP_MAX_SEC (기본 30.0)

초과 세그는 균등분할. 강제분할 위치는 콘텐츠와 무관한 임의 지점 — 서로 다른 실행·설정
간 경계 비교 시 이 지점들은 원래 안 맞는 게 정상이다.

### PREP_DETECT_FPS (기본 0)

분할 검사 밀도(초당 검사 프레임 수). 0=전 프레임(최고 정밀). 내부적으로
`frame_skip = round(원본fps ÷ detect_fps) − 1` 환산 — 소스 fps 가 달라도 시간 밀도 동일.

- **주의**: 값을 낮추면 짧은 장면 통누락·점수 왜곡으로 결과가 크게 변한다
  (실측: detect_fps=2 에서 세그 1678→941, 44% 소실). 속도가 필요하면 이것보다
  **PREP_DETECT_WORKERS 병렬을 먼저** 쓸 것 — 결과 왜곡 없이 빨라진다.

### PREP_DETECT_WORKERS (기본 0, 운영 12)

시간축 청크 병렬 탐지 프로세스 수(0·1=단일 패스).

## 병렬 탐지 설계 (이음새 문제 해결)

naive 한 "K등분해서 각자 분할"은 **청크 이음새마다 가짜 경계**가 생긴다. 이 구현은:

1. 청크는 '컷 후보'가 아니라 **탐지 분담 구간** — 각 워커는 자기 구간 ± 패딩
   (`max(2초, 2×min_sec)`)을 탐지해 문맥을 확보한다.
2. 워커는 반개구간 `(start, end]` 안에서 **실제 감지된 컷 시각만** 반환 — 인접 청크와
   중복 없음, 이음새는 반환값에 등장할 수 없음.
3. 전 워커 컷의 **합집합**으로 장면 목록을 재구성 → 기존 후처리(_postprocess) 재사용.

검증(115분 실영상): 단일 패스와 세그 수 동일(1678), 이음새 11곳 중 가짜 경계 0,
차이는 청크 시작점 탐지기 워밍업으로 인한 ±1초 재배열 극소수.

- 프로세스는 **spawn** 컨텍스트(opencv fork 비안전 회피). 워커 함수는 모듈 최상위
  (picklable). 테스트 스크립트에서 호출할 때는 `if __name__ == "__main__":` 가드 필수.
- 워커 수 실측(48 vCPU): 12워커 13.3s, 42워커 11.3s — **12 근처에서 포화**
  (워커당 spawn·임포트·seek 고정비 1~2초가 바닥을 만듦). 물리 코어 수 이상은 무의미.

## 후처리(_postprocess) 세부

```
float 장면 목록
→ min_sec 미만 병합(직전 흡수, 첫 세그는 다음이 흡수)
→ max_sec 초과 균등분할 (n = ceil(dur/max_sec))
→ 끝점들을 정수 초로 스냅해 {0, ..., total} 경계점 집합 구성
→ 인접 경계점 쌍 → (seg_id(1부터), start, end) — 틈·겹침 없음 보장
```

정수 스냅이 최종 방어선: 탐지 정밀도의 ±0.5초 미만 오차는 최종 타일에 나타나지 않는다.

## 품질 검증 이력 (2026-07-20, 115분 야구 중계)

- 기존 검증본(t_segment) 대비 ±1초 경계 재현율 76%, 불일치 지점 22곳 육안 대조:
  일치 경계 4/4 실제 컷, 신규 전용 7/9 실제(2 과분할: 동일 샷 모션·팬),
  기존 전용 6/9 실제(3 과분할: 투구 모션·줌·포커스). **양쪽 품질 대등** — 차이는
  빠른 컷 구간에서 서로 다른 위치를 고른 것이 대부분.
- 신규가 놓친 유형: 디졸브 1건(기존은 포착) — ContentDetector 특성.
